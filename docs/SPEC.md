## 1. 시스템 개요

### 1.1 목적
한국투자증권 오픈API를 활용한 2트랙 모멘텀 전략(섹터로테이션 + 단기스윙)의 스크리닝·매매관리·성과추적을 하나의 웹 대시보드에서 수행한다. UI는 Excel 스프레드시트 스타일로 설계하여 직관적이고 익숙한 사용 경험을 제공한다.

### 1.2 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 프레임워크 | Next.js 14 (App Router) + TypeScript | 서버 컴포넌트 + API Routes |
| 스타일링 | Tailwind CSS 또는 인라인 스타일 | Excel 스프레드시트 테마 |
| DB | Supabase (PostgreSQL) | 매매일지 + 스크리닝 이력 |
| 배포 | Vercel (무료 티어) | 서버리스 함수 |
| 설치 | PWA (manifest + service worker) | 홈화면 앱 추가 |

### 1.3 아키텍처

```
[브라우저 — MomentumSheet]
        ↓ fetch (CSR)
[Next.js API Routes — /api/*]
   ├─ /api/auth/token     → 한투 OAuth 토큰 발급/캐싱
   ├─ /api/balance         → 한투 잔고조회 (TTTC8434R)
   ├─ /api/balance/refresh → 보유종목 현재가 수동 새로고침
   ├─ /api/swing/screen    → 시세 API × 20종목 → 스코어링
   ├─ /api/swing/pool2     → 거래량순위 + 신고근접 → 2차 풀 수집
   ├─ /api/sector/screen   → 기간별시세 API × 7종목 → 복합모멘텀
   ├─ /api/journal         → Supabase CRUD (매매일지)
   ├─ /api/journal/stats   → 매매 성과 통계 (차트용 집계)
   ├─ /api/history         → Supabase CRUD (스크리닝 이력)
   ├─ /api/history/swing   → 스윙 주차별 이력 조회
   └─ /api/history/sector  → 섹터 월별 이력 조회
```

### 1.4 환경변수

| 변수명 | 용도 |
|--------|------|
| `KIS_APP_KEY` | 한투 앱키 |
| `KIS_APP_SECRET` | 한투 시크릿 |

## 2. 공통 모듈

### 2.1 한투 API 토큰 관리 (`lib/kis-auth.ts`)

**기능**: OAuth2 토큰 발급 및 Supabase DB 저장

| 항목 | 상세 |
|------|------|
| 엔드포인트 | `POST /oauth2/tokenP` |
| 요청 바디 | `{ grant_type: "client_credentials", appkey, appsecret }` |
| 응답 | `{ access_token, token_type, expires_in }` |
| 저장 | Supabase `kis_token` 테이블 (인메모리 캐시 사용 안 함) |
| 조회 순서 | DB 조회 → 만료 시 새 발급 |
| 무효화 | DB의 `expires_at`을 과거로 설정 |
| 에러 처리 | 발급 실패 시 에러 반환, EGW00123 시 재시도 |

```typescript
export async function getToken(): Promise<string>
export async function wasTokenRecentlyIssued(withinMs?: number): Promise<boolean>
export async function invalidateToken(): Promise<void>
```

### 2.2 한투 API 호출 공통 함수 (`lib/kis-api.ts`)

**기능**: 공통 헤더 구성 + 호출 + 에러 핸들링

| 항목 | 상세 |
|------|------|
| Base URL | `https://openapi.koreainvestment.com:9443` |
| 공통 헤더 | `content-type`, `authorization` (Bearer), `appkey`, `appsecret`, `tr_id`, `custtype: "P"` |
| Rate Limit | 초당 20회 제한. 호출 간 최소 50ms 간격 (큐 방식) |
| 에러 처리 | HTTP 에러 → 재시도 1회 / API 에러코드 → 로그 후 반환 |

```typescript
export async function kisGet(path: string, trId: string, params: Record<string, string>): Promise<any>
export async function kisPost(path: string, trId: string, body: Record<string, any>): Promise<any>
```

### 2.3 API 호출 제한 관리 (`lib/rate-limiter.ts`)

| 항목 | 상세 |
|------|------|
| 방식 | 토큰 버킷 (초당 20회) |
| 구현 | 큐에 요청을 넣고 50ms 간격으로 순차 실행 |
| 타임아웃 | 큐 대기 10초 초과 시 에러 |

### 3.6 오늘의 할일 체크리스트

계좌 요약 위에 표시. 전략 규칙 + 보유종목 상태를 기반으로 자동 생성된다.

**할일 자동 생성 규칙**:

| 조건 | 할일 항목 | 아이콘 | 색상 |
|------|----------|--------|------|
| 오늘 = 금요일 AND 이번주 스윙 스크리닝 미실행 | "스윙 스크리닝 실행 필요" | `□` | 기본 |
| 오늘 = 월말 마지막 거래일 AND 이번달 섹터 스크리닝 미실행 | "섹터 스크리닝 실행 필요" | `□` | 기본 |
| 오늘 = 월 첫 거래일 AND 섹터 매수 미완료 | "섹터 매수 실행 필요" | `□` | 기본 |
| 오늘 = 월요일 AND 스윙 매수 미완료 | "스윙 매수 실행 필요" | `□` | 기본 |
| 보유종목 수익률 ≥ 익절목표 - 1% | "종목명 +N% → 익절 임박" | `⚠` | `#bf8f00` (경고 노랑) |
| 보유종목 수익률 ≤ 손절목표 + 1% | "종목명 -N% → 손절 임박" | `⚠` | `#9c0006` (경고 빨강) |
| 오늘 = 금요일 AND 스윙 보유종목 미청산 | "스윙 종가청산 확인 필요" | `⚠` | `#9c0006` |
| 오늘 = 월말 AND 섹터 보유종목 미청산 | "섹터 종가청산 확인 필요" | `⚠` | `#9c0006` |
| 스크리닝 완료 | "N주차 스크리닝 완료 (종목명)" | `✓` | `#006100` (초록) |
| 매수 완료 | "매수 완료 (종목명)" | `✓` | `#006100` |

**익절/손절 임박 판단 기준**:

| 전략 | 익절 임박 | 손절 임박 |
|------|----------|----------|
| 스윙 | 수익률 ≥ +6% (목표 +7%) | 수익률 ≤ -2% (목표 -3%) |
| 섹터 | 수익률 ≥ +6% (목표 +7%) | 수익률 ≤ -4% (목표 -5%) |

**데이터 소스**:
- 보유종목 + 수익률: `/api/balance` 응답
- 스크리닝 완료 여부: `screening_history` 테이블에서 현재 주차/월 조회
- 매수 완료 여부: `journal` 테이블에서 현재 주차/월 미청산 기록 조회
- 요일/날짜 판단: 클라이언트 로컬 시간 (KST)

**할일이 없을 때**: "오늘 할 일이 없습니다 ✓" 표시

### 3.7 보유종목 수동 새로고침

**트리거**: 계좌 요약 섹션 헤더 우측의 `🔄 시세 새로고침` 버튼

| 항목 | 상세 |
|------|------|
| API | `GET /api/balance/refresh` → 한투 잔고조회 API 재호출 |
| 버튼 스타일 | Excel 리본 버튼 스타일 (`#f3f3f3` 배경, `#c0c0c0` 테두리) |
| 로딩 상태 | 버튼 텍스트 "갱신 중..." + 아이콘 회전 |
| 완료 | 테이블 데이터 갱신 + 상태바에 "갱신 완료 (HH:MM)" 표시 |
| 장 마감 후 | 정상 작동 (마감가 기준 데이터 반환) |
| Rate Limit | 최소 5초 간격 제한 (연타 방지) |

**자동 새로고침은 없음** — 사용자가 버튼을 눌렀을 때만 시세 갱신

### 3.8 동작 흐름

1. 페이지 로드 시 `GET /api/balance` 호출 (잔고 + 보유종목)
2. 할일 체크리스트 생성 (잔고 데이터 + Supabase 이력 기반)
3. 로딩 중: 셀에 "로딩 중..." 표시
4. 응답 수신: 할일 + 계좌 요약 + 보유종목 테이블 렌더링
5. 에러 시: 셀에 "#ERROR - 연결 실패" 표시 (Excel 에러 스타일)
6. `🔄 시세 새로고침` 클릭 시: 잔고 재조회 → 테이블 + 할일 갱신

### 3.9 조건부 서식 규칙

### 4.4 스코어링 시스템

**필요한 API 호출 (종목당)**:

| API | TR_ID | 용도 | 주요 응답 필드 |
|-----|-------|------|---------------|
| 현재가 조회 | `FHKST01010100` | 현재가, 52주 고가 | `stck_prpr`, `stck_hgpr` (52주고가) |
| 일별시세 | `FHKST01010400` | 최근 30일 종가·시가·거래량 | `stck_clpr`, `stck_oprc`, `acml_vol` |
| 투자자별 | `FHKST01010900` | 외국인 순매수 | `frgn_ntby_qty` |

**원시 지표 계산**:

| # | 지표 | 계산식 |
|---|------|--------|
| 1 | 거래량비율 (`volRatio`) | 최근 5일 평균 거래량 ÷ 최근 20일 평균 거래량 |
| 2 | 52주고가 근접률 (`highRatio`) | 현재가 ÷ 52주고가 × 100 |
| 3 | 5일선 이격 (`ma5Gap`) | (현재가 - MA5) ÷ MA5 × 100 |
| 4 | 정배열 (`isAligned`) | MA5 > MA20 → true/false |
| 5 | 20일선 기울기 (`slope`) | (오늘 MA20 - 5일전 MA20) ÷ 5일전 MA20 × 100 |
| 6 | 외국인 순매수 일수 (`foreignDays`) | 최근 5일 중 `frgn_ntby_qty > 0`인 일수 |
| 7 | 연속 양봉 일수 (`bullDays`) | 최근 5일 중 종가 > 시가인 일수 |
| 8 | 이격도 (`gapRatio`) | 현재가 ÷ MA20 × 100 |

**MA 계산**:
- MA5 = 최근 5일 종가 평균
- MA20 = 최근 20일 종가 평균

**점수 변환 규칙 (총 100점)**:

| 항목 | 배점 | 구간별 점수 |
|------|------|------------|
| 거래량 강도 | /15 | ≥2.0→15, ≥1.5→12, ≥1.2→9, ≥0.8→6, else→0 |
| 52주고가 근접 | /15 | ≥95%→15, ≥90%→12, ≥85%→9, ≥80%→6, else→0 |
| 5일선 이격 | /10 | 0~3%→10, 3~5%→7, 5~7%→4, else→0 |
| 정배열 | /10 | MA5>MA20→10, else→0 |
| 20일선 기울기 | /10 | ≥1%→10, ≥0.5%→7, ≥0%→4, else→0 |
| 외국인 수급 | /15 | 5일→15, 4일→12, 3일→9, 2일→6, else→0 |
| 연속 양봉 | /10 | 일수×2 (최대 10) |
| 이격도 적정성 | /15 | 100~105%→15, 105~108%→10, 97~100%→7, else→0 |

### 5.6 복합 모멘텀 계산

```
1M 수익률 = (현재가 - 1M전 종가) / 1M전 종가 × 100
3M 수익률 = (현재가 - 3M전 종가) / 3M전 종가 × 100
복합점수 = (1M수익률 × 0.6) + (3M수익률 × 0.4)
```

**예외 처리**: 상장 후 3개월 미만 → 3M 데이터 없음 → 1M 수익률만 사용 (복합점수 = 1M수익률)

### 5.7 RSI(3) 진입 필터 (`lib/rsi.ts`)

모멘텀 1위 ETF 선정 후, RSI(3) 눌림목 확인 후 진입하여 고점 진입 → 즉시 손절 위험을 줄인다.

**RSI 계산 스펙**:

| 항목 | 상세 |
|------|------|
| 데이터 소스 | `FHKST03010100` output2 (일별 종가, 최신순) |
| 기간 | RSI(3) — period=3 |
| 안정화 버퍼 | period × 2 + 14 = 20일치 필요 |
| 스무딩 | Wilder's Smoothing |
| 반환값 | 0~100 (소수점 1자리), 데이터 부족 시 null |

| `NO_DATA` | RSI 계산 불가 | 데이터 부족 |

1. "섹터 스크리닝" 버튼 클릭
2. `GET /api/sector` → 7개 ETF 기간별시세 조회 + RSI(3) 계산
   - 종목당 1회 API 호출 → 총 7회
   - RSI 계산은 기존 output2 데이터 재사용 (추가 API 호출 없음)
3. 복합점수 내림차순 정렬 + 1위 ETF를 진입 대상으로 확정
4. 결과 테이블 + 진입 판단 카드 렌더링
5. Supabase 자동 저장 (upsert)

**월중 RSI 새로고침**:

1. "RSI 새로고침" 버튼 클릭
2. `GET /api/sector/rsi?code={1위ETF코드}` → 1건만 API 호출
3. RSI(3) 재계산 → 진입 판단 카드만 업데이트
4. 스크리닝 테이블 및 순위는 변경되지 않음

```typescript
  summary: {
    totalTrades: number;       // 청산 완료 거래 수
    wins: number;
    losses: number;
    winRate: number;           // 승률 (%)
    totalPnl: number;          // 총 실현손익 (원)
    avgPnlPerTrade: number;    // 건당 평균 손익
    avgWinAmount: number;      // 평균 수익 (승리 건)
    avgLossAmount: number;     // 평균 손실 (패배 건)
    profitFactor: number;      // 총수익 / 총손실
    maxWin: number;            // 최대 수익 건
    maxLoss: number;           // 최대 손실 건
    avgHoldingDays: number;    // 평균 보유일
  };

  // 전략별 비교
  byStrategy: {
    swing: { trades: number; winRate: number; totalPnl: number; avgPnl: number; };
    sector: { trades: number; winRate: number; totalPnl: number; avgPnl: number; };
  };

  // 시계열 (차트용)
  timeline: {
    label: string;            // "2026-W09" 또는 "2026-02"
    cumulativePnl: number;    // 누적 손익
    trades: number;           // 해당 기간 거래 수
    winRate: number;          // 해당 기간 승률
  }[];

  // 청산 사유별
  byReason: {
    reason: string;           // 익절/손절/종가청산
    count: number;
    totalPnl: number;
    avgRate: number;
  }[];
}
```

### 7.6 성과 요약 카드 항목

| 지표 | 계산식 | 서식 |
|------|--------|------|
| 총 거래 | 청산 완료 건수 | 정수 |
| 승률 | 승 ÷ 총거래 × 100 | %, 50% 이상=초록, 미만=빨강 |
| 총 손익 | SUM(profit_loss) | 원, 조건부서식 |
| 건당 평균 | 총손익 ÷ 총거래 | 원, 조건부서식 |
| 손익비 (Profit Factor) | 총수익금 ÷ \|총손실금\| | 소수점 2자리, ≥1.5 초록 |
| 평균 보유일 | AVG(sell_date - buy_date) | 소수점 1자리 |
| 최대 수익 | MAX(profit_loss) | 원 |
| 최대 손실 | MIN(profit_loss) | 원 |

### 7.7 동작 흐름

1. 페이지 진입 시 `GET /api/journal/stats` 호출 (기본: 전체 기간, 전체 전략)
2. 성과 요약 + 전략별 비교 + 차트 + 청산 사유 테이블 렌더링
3. 기간/전략 필터 변경 시: 쿼리 재호출 → UI 갱신
4. 거래 0건일 때: "아직 청산 완료된 거래가 없습니다" 표시

## 10. 에러 처리 패턴

| 상황 | UI 표시 | 비고 |
